# Systema Naturae

Interactive taxonomy portal. Each family lives in its own directory; the portal aggregates them at build time.

> Navigation architecture documented in [`docs/navigation.md`](docs/navigation.md).

## Commands

```bash
cd portal && npm run dev          # rebuild unified taxonomy, start dev server
cd portal && npm run build        # rebuild unified taxonomy, production build
cd portal && npm run typecheck    # TypeScript check
cd portal && sh scripts/buildData.sh   # rebuild unified-taxonomy.json only (fast verify; auto-detects bun vs tsx)
```

## Graph UI features

Portal currently contains **54,977 physical nodes + 128,352 compressed species = 183,329 nodes represented**, totalling **165,138 species** across **1,114 families** in **19 classes** spanning **7 phyla** (Chordata, Arthropoda, Mollusca, Echinodermata, Tardigrada, Cnidaria, Ctenophora). All 254 IOC bird families are represented. The Coverage modal (ⓘ button) shows per-class statistics with sort/filter controls and a Growth timeline chart with node/species history.

> **Node count math:** `buildData.ts` runs a compression step — minimal SPECIES nodes (no description or generic "a species in the genus X" placeholder, no children) under a GENUS get collapsed into a sibling-free `speciesList[]` array on that GENUS instead of remaining as physical tree nodes. Counting only `n.rank === 'SPECIES'` undercounts by ~128k. Add `speciesList` entries to get the true species total.

### Node sizing
`nodeR()` in `FamilyTree.tsx` uses widened radii: family (12), subfamily (9), genus (6.5), species (2.5). Provides clearer hierarchy distinction.

### Family focus pruning
When a family is focused in the tree, all non-focused families' children are pruned to single dots. The focused family gets the full circle naturally without arc remapping.

### Genus rank badge
`UnifiedInfoPanel.tsx` shows a `GENUS` rank badge with accent colour in `GenusPanel`.

### Coverage modal
The ⓘ Coverage button opens a modal with sort/filter controls (Gaps first / A–Z; All / Complete / Partial / Gaps), per-class mini progress bars, family status tags, and a Growth tab showing a D3 chart of cumulative species and node count over time with interactive tooltips and togglable series.

### Wheel of Nature, Statistics Header, Book View, Species of the Day
All shipped — see `portal/src/components/` and `docs/backlog.md` for status notes.

## Adding a new family — checklist

1. **`<class>/<order>/<family>/src/data/<family>.json`** — species data.  
   Root: `{ id, name, rank: "FAMILY", commonName, children: [...] }`  
   Node fields: `lineage`, `continents[]`, `subspeciesCount`, `description`, `namedAfter`, `sourcedFrom`

2. **`portal/data/taxonomy.json`** — add a FAMILY node with `appSlug` and `speciesCount` (real-world total, used by Coverage modal) inside the correct ORDER/CLASS.

3. **`portal/src/colorRegistry.ts`** — add a `ColorTheme` constant and register it in `COLOR_REGISTRY` under the `appSlug` key.

4. Run `cd portal && sh scripts/buildData.sh` to verify — warns on missing data files, prints total node count.

5. **Check `shared/data/pending-eponyms.json`** — if the new family's `familySlug` appears there, add those species with their `namedAfter` field and remove the entry.

6. **`portal/data/species-news.json`** — add a `family_added` event for the new family with `date` set to today, so the NewsBell in the header shows the addition.

7. **Regenerate gap reports** so the new family appears in tracking:
   ```bash
   cd portal
   npx tsx scripts/findGaps.ts
   npx tsx scripts/reportPhyla.ts
   npx tsx scripts/generateGapTasks.ts
   npx tsx scripts/buildImportLog.ts
   ```

## How the graft works

`buildData.ts` walks `taxonomy.json`, tracking the current CLASS and ORDER as it descends. When it hits a FAMILY node with `appSlug`, it reads `<root>/<class>/<order>/<appSlug>/src/data/<appSlug>.json`, stamps every descendant node with `familySlug`, `className`, `orderName`, and `extinct` (detected from `†` prefix or "extinct"/"fossil" in description), and splices the children in. Tardigrada (no CLASS ancestor) resolves to `<root>/tardigrada/…`.

After grafting, a **compression pass** walks every GENUS: minimal SPECIES children (no real description, no children of their own) move into a flat `speciesList[]` on the genus, dropping their tree weight. `colorRegistry.ts` uses `familySlug` to look up the `ColorTheme` for rendering.

## Key TaxonNode fields

| Field | Where set | Purpose |
|---|---|---|
| `namedAfter` | family JSON | Person species is named after; shown in tree tooltip and Eponyms modal |
| `familySlug` | buildData.ts | Stamped at build time; used for color lookup and navigation |
| `className`, `orderName` | buildData.ts | Stamped at build time; used by Coverage modal and panels |
| `extinct` | buildData.ts | Auto-detected from `†` prefix or "extinct"/"fossil" in description |
| `sourcedFrom` | importer | `"wikipedia"`, `"gbif"`, `"ollama"`, etc. Used by OptionsPanel highlight toggle |
| `iucnStatus` | wiki-images.json merge | IUCN red-list status code (e.g. `LC`, `EN`, `CR`) |
| `speciesCount` | taxonomy.json | Real-world species total for the family (not portal node count) |
| `continents[]` | family JSON | Either continent codes (`"NA"`, `"EU"`) or full names — both accepted |

## Sidecar data: `shared/data/wiki-images.json`

Keyed by scientific name. ~148k entries; ~34k carry an `image` URL, ~33k a `commonsCat`, ~31k an `iucnStatus`, ~10k `rangeMaps`. Populated by `scripts/fetchWikidataImages.ts` from Wikidata SPARQL. Consumed at runtime by `UnifiedInfoPanel` and `ImageTabs`. Kept out of `unified-taxonomy.json` to keep the bundled tree small.

## Offline Wikipedia pipeline (Macie only)

A 2026-06-01 enwiki dump lives on external volumes — see global `~/.config/opencode/AGENTS.md` for paths. The pipeline:

1. **`scripts/buildWikipediaDb.py`** parses the raw multistream dump into `/Volumes/WikiDump/wiki-pages.sqlite` (title → wikitext). Use `tailWikiDbProgress.py` to watch progress; `watchWikipediaDump.sh` waits for the volume.
2. **`scripts/mergeWikiInfoboxes.ts`** parses infoboxes out of the SQLite DB and merges descriptions/continents/IUCN into family JSONs.
3. **`scripts/enrichFromWikipedia.ts`** (root) uses the SQLite DB when mounted, falling back to the Wikipedia REST API otherwise. `portal/scripts/enrichFromWikipedia.ts` is the REST-API-only sibling.

`52,655 species` currently carry `sourcedFrom: "wikipedia"` — toggle the "Highlight Wikipedia species" checkbox in the ⚙ OptionsPanel to see them.

## Gap tracking

- `portal/scripts/findGaps.ts` → `portal/data/gap-report.json` (per-family `have` / `speciesCount` / `gap`).
- `portal/scripts/reportPhyla.ts` → `docs/gap-tasks-phyla.md` (phylum/class roll-up). Also POSTs to `localhost:9876` if `phylumProgressd.ts` is running.
- `portal/scripts/generateGapTasks.ts` → `docs/gap-tasks.md` (per-family TODO list).
- `portal/scripts/phylumProgressd.ts` — long-running daemon that records each POST so progress is graphable over time.

Always run these three after a significant import.

## Fixing sanity issues

`portal/data/sanity-queue.json` is currently `[]` — all 4057 historical issues across 66 families have been resolved (full report in `sanity-issues.json`). Fix scripts live in `scripts/`:

| Script | Fixes |
|---|---|
| `fix_continents.py` | Replaces abbreviated continent codes (`"EU"` → `"Europe"`) with full names |
| `fix_duplicates.py` | Removes duplicate species entries |
| `fix_empty_genera.py` | Removes genus nodes with no children |
| `fix_lineage.py` | Infers missing `lineage` field from parent genus |
| `fix_subspecies.py` | Adds `subspeciesCount: 0` to species nodes missing it |
| `fix_all_descriptions.py` | Generates descriptions for all species based on lineage/genus |

**Workflow for a new sanity scan:**
1. Run each fix script from repo root
2. Rebuild and verify: `cd portal && sh scripts/buildData.sh`
3. Update `sanity-queue.json` to reflect remaining issues
